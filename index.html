    📌 성능 차이 분석 (scanner.nextLine() vs scanner.nextInt())

    결과 요약

    코드메모리 사용량실행 시간
    코드 1 (scanner.nextLine())17624KB168ms
    코드 2 (scanner.nextInt())17700KB180ms

    ✅ 분석 결과:

    • scanner.nextLine()을 사용한 코드가 더 빠름 (168ms vs 180ms)
    • 메모리 사용량도 scanner.nextLine()이 더 적음 (17624KB vs 17700KB)
    • 입력을 한 번만 받는 것이 더 효율적이라는 예측이 맞음! 🚀

    📌 왜 scanner.nextLine()이 더 빠를까?

    1️⃣ scanner.nextInt()는 여러 번 호출되면서 추가적인 처리 비용이 발생

    • 코드 1 (scanner.nextLine())
      String[] userInput = scanner.nextLine().split(" ");
      • 한 번의 입력 (nextLine())으로 문자열 전체를 가져옴.
      • 이후 split(" ")을 사용해 배열로 변환.
      • I/O 연산이 한 번만 발생 → 성능 이점.
    • 코드 2 (scanner.nextInt())
      int a = scanner.nextInt();
int b = scanner.nextInt();
      • scanner.nextInt()를 두 번 호출 → 입력을 두 번 받음.
      • 내부적으로 nextInt()는 공백을 무시하는 작업 및 타입 변환을 추가적으로 수행.
      • 이 과정에서 여러 번의 I/O 연산이 발생 → 실행 시간이 증가.

    2️⃣ scanner.nextInt()는 내부적으로 nextLine()을 호출하고 추가 처리

    • scanner.nextInt()는 입력을 정수로 변환할 때, 내부적으로 입력 버퍼를 처리하는 과정이 필요함.
    • 하지만 scanner.nextLine()은 그냥 한 번에 문자열을 가져오므로 I/O 연산이 적게 발생.
    • 따라서 한 번에 입력을 받은 후, split()으로 나누는 것이 더 빠름.

    📌 결론: 언제 nextLine()과 nextInt()를 사용할까?

    상황추천 입력 방식
    한 줄 입력을 받아 숫자로 변환해야 할 때scanner.nextLine() + split() (더 빠름)
    입력이 여러 줄로 주어질 때scanner.nextInt() (여러 줄 입력 시 유용)
    공백을 포함한 문자열 입력이 필요할 때scanner.nextLine() (문자열 전체 입력 가능)

    📌 개선 가능한 점

    🚀 BufferedReader + StringTokenizer 사용

    • Scanner는 편리하지만, 속도가 상대적으로 느림.
    • BufferedReader + StringTokenizer 조합을 사용하면 훨씬 더 빠르게 처리 가능.

    📌 개선된 코드 (더 빠름)

    import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStreamReader;
import java.util.StringTokenizer;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st = new StringTokenizer(br.readLine());

        int a = Integer.parseInt(st.nextToken());
        int b = Integer.parseInt(st.nextToken());

        System.out.println(a - b);
    }
}

    🚀 성능 예상

    코드예상 실행 시간예상 메모리
    Scanner + nextLine()168ms17624KB
    Scanner + nextInt()180ms17700KB
    BufferedReader + StringTokenizer🚀 100~120ms💡 더 적음

    ✅ BufferedReader가 Scanner보다 빠른 이유

    • Scanner는 내부적으로 정규식을 사용하여 입력을 파싱하는데, 이 과정이 느림.
    • BufferedReader는 단순한 문자열 입출력만 처리하여 훨씬 빠름.
    • StringTokenizer는 split(" ")보다 빠르게 문자열을 분리.

    📌 최종 결론

    ✔ 입력을 한 번만 받는 scanner.nextLine() 방식이 더 빠름 (168ms vs 180ms).

    ✔ scanner.nextInt()는 내부적으로 버퍼를 처리하는 과정이 있어 더 느림.

    ✔ 성능 최적화를 위해 BufferedReader + StringTokenizer를 사용하는 것이 좋음.

    🚀 즉, 빠른 입력 처리가 필요할 때는 BufferedReader + StringTokenizer를 적극 활용하자! 🚀
